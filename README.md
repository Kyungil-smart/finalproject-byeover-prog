# AprilLog

정렬 퍼즐로 스킬을 발동해 몰려오는 몬스터를 막아내는 **퍼즐 디펜스** 모바일 게임.
아래에서는 책장을 정리하고, 위에서는 그 정리가 마법이 되어 터진다.

- 엔진: Unity 6000.3.11f1 (URP 2D)
- 플랫폼: Android (세로, 기준 해상도 1440x3120)
- 백엔드: Firebase Authentication / Cloud Firestore
- 언어: 한국어 / 영어

---

## 게임 핵심 루프

1. 화면 하단의 퍼즐 보드에서 같은 색 유닛(책)을 드래그해 한 줄을 완성한다.
2. 정렬이 완성될 때마다 캐릭터가 공격을 발사하고, 콤보가 쌓인다.
3. 색 조합 레시피를 완성하거나 콤보가 조건에 도달하면 강력한 원소 스킬이 발동한다.
4. 몬스터를 처치해 얻은 경험치로 전투 중 레벨업하면 인챈트(스킬 카드)를 선택해 빌드를 만든다.
5. 웨이브를 버티고 보스를 처치하면 스테이지 클리어. 보상으로 캐릭터와 아티팩트를 성장시킨다.

---

## 인게임 기능

### 전투
- **정렬 공격**: 퍼즐 한 줄 완성 시 해당 색의 공격/스킬 발사
- **자동 공격**: 일반 스킬 인챈트 획득 시 활성화되는 주기 공격 (기본 공격의 30% 위력, DB 정의)
- **조합 스킬**: 색 조합 레시피 완성 시 발동. 한 번의 정렬로 여러 레시피가 동시에 완성되면 전부 발사
- **콤보 스킬**: 콤보 수가 스킬별 배수에 도달할 때마다 발동
- **조커**: 쿨타임제 보드 정리기 — 테이블을 순회하며 유닛을 자동 정렬

### 원소 스킬 25종 (5속성 x 5종, 각 3레벨)

| 속성 | 스킬 |
|---|---|
| 불 | 파이어브레스 / 화염 작렬 / 화염 정령 소환 / 대지 균열 / 메테오 |
| 물 | 물 폭탄 / 탄환 세례(물대포) / 급류 / 파도 소환 / 하이드로 펌프 |
| 바람 | 헤이스트 / 바람 칼날 / 돌풍 / 허리케인 / 템페스트 |
| 번개 | 구형 번개 / 에너지볼 / 방전 / 벼락 / 뇌격 |
| 얼음 | 마칭 아이스 / 글레이셜 피어스 / 빙결 지대 / 얼음 결정 / 절대영도 |

- 형태별 알고리즘: 투사체(관통/멀티히트), 장판(배치 전략 x 펄스), 이동형(전진 판정), 연쇄(튕김)
- 상태이상(CC): 슬로우 / 넉백 / 스턴 / 속박
- 데미지, 판정 범위, 발사 수 등 수치는 전부 데이터 시트에서 읽는다 (코드 수정 없이 밸런싱 가능)

### 인챈트 (전투 중 성장)
- 전투 중 레벨업 시 카드 3장 중 선택. 리롤 지원
- 스킬 인챈트(신규 스킬 획득/강화)와 스탯 인챈트로 구분
- 보유 슬롯 초과 시 교체 팝업 (일시정지 상태에서 선택)
- 리세마라 방지: 카드가 화면에 뜨는 순간 결과를 저장 — 강제종료 후 재진입해도 같은 카드가 복원됨

### 스테이지 / 웨이브
- 테마 2종 x 챕터 5개 x 스테이지 7개 구성 (+ 튜토리얼 전용 챕터)
- 웨이브 규칙 기반 스폰: 시간 종료 / 보스 처치 종료 조건, 특수 웨이브(러시 / 엘리트 / 보스)
- 몬스터 스테이지 스케일링: 진행도에 따라 체력/공격/방어 성장
- 전투 도중 이탈 시 이어하기 (퍼즐 보드, 인챈트, HP, 진행 웨이브까지 복원)

### 튜토리얼
- 최초 실행 시 인트로 시나리오 후 바로 전투로 진입하는 시나리오형 튜토리얼
- 필패 연출 전투 → 로비 성장 가이드(레벨업, 아티팩트) → 재진입 보스 클리어의 기승전결 구조
- 단계 데이터(ScriptableObject) 기반이라 순서/문구를 데이터로 수정 가능

## 아웃게임 기능

### 로비
- 하단 탭 페이지 전환: 메인 로비 / 캐릭터 성장 / 인챈트 도감 / 콘텐츠 / 하우징 / 상점
- 챕터 카드를 넘겨 선택하고 스태미나를 소모해 진입. 최초 진입 챕터는 스토리를 먼저 재생
- 저장된 전투가 있으면 스태미나 소모 없이 이어하기

### 캐릭터 성장
- 골드 + 양피지로 캐릭터 레벨업
- 최대 HP / 공격력 / 효과력 / 관통 성장 보너스가 인게임 스탯에 그대로 적용

### 아티팩트
- 가챠: 1회/10회 뽑기, 상자별 소모 재화 상이 (골드/양피지/다이아), 광고 무료 뽑기
- 천장(확정 레전더리)과 마일리지(누적 보상) 시스템
- 장착 슬롯 3개, 강화(골드+강화석) / 돌파(중복 기어 or 조각) / 분해 / 제작(조각 5개)

### 하우징
- 가구 구매/배치 (장식/배경/기능 카테고리)
- 방치 보상: 시간당 재화 적립 (클리어 챕터에 따라 보상량 증가), 광고 보상
- 가구 상호작용 연출 (침대/주방/책상/소파), 책장에서 스토리 다시보기 진입

### 스토리
- 챕터 최초 진입 시 자동 재생, 감상 기록은 클라우드 저장
- 하우징 책장의 "서사 보관소"에서 해금된 에피소드 다시보기

### 시스템
- **스태미나**: 챕터 진입 시 소모, 시간 회복(오프라인 회복 포함), 튜토리얼 중 면제
- **재화**: 골드(70001) / 양피지(70002) / 다이아(70003) / 강화석(70004) / 레전더리 조각(70005) / 행동력(10001)
- **정산 보상**: 전투 중 누적(웨이브/엘리트/보스 킬) + 클리어 변동 보상(최초 1회 / 반복, 진행도 비례 성장)
- **클라우드 저장**: 진행도, 재화, 아티팩트, 하우징, 성장, 설정 전부 Firestore 동기화. Firebase 미가용 시 오프라인 모드 + 로컬 백업, 네트워크 복구 시 자동 동기화
- **로그인**: Firebase 게스트(익명) / Google 로그인
- **사운드**: BGM/SFX 분리 채널, 35클립 사운드 라이브러리, 전 버튼 클릭음 자동 바인딩, 볼륨 설정 저장
- **로컬라이징**: 한국어/영어, 기기 언어 자동 감지, 런타임 즉시 전환

## 아키텍처

**"씬이 아니라 코드가 조립한다"** — 9인이 씬을 동시에 작업해도 깨지지 않는 것이 설계 목표.

```
_Boot (영속 싱글톤: DataManager / GameManager / AudioManager / SceneTransition)
  └─ 씬 진입 → Bootstrap이 의존성 순서로 조립
       Repository (시트→SO) → Model (상태·규칙) → System (전투·스폰) → View (HUD·팝업)
  └─ 런타임 통신은 이벤트만 (직접 참조 최소화, 참조가 비면 자동 탐색으로 자가 복구)
```

- **도메인당 입구 하나**: 재화는 GameManager, 아티팩트 장착은 ArtifactManager, 인게임 팝업/일시정지는 ScreenNavigator만 통과
- **데이터 주도**: 게임 수치는 구글 시트 → JSON → ScriptableObject 파싱. 코드는 규칙만 알고 숫자는 모른다
- **오브젝트 풀링**: 몬스터/투사체 전량 풀링, 핫패스 무할당 원칙

## 프로젝트 구조

```
AprilLog/Assets/
├─ _Game/            게임 코드 (Core / Features / UI / Data)
├─ _Project/         데이터 SO, 기획 문서
├─ Resources/        런타임 로드 에셋 (테이블 JSON, VFX/사운드 라이브러리)
├─ Scenes/           _Boot → _Lobby → _Story → _InGame (+_Test)
└─ Imports/          외부 에셋 (git 미추적 — 팀 공유 채널로 배포)
```

## 씬 구성

| 씬 | 역할 |
|---|---|
| _Boot | 초기화, 로그인, 클라우드 데이터 로드 |
| _Lobby | 로비 전 페이지 (성장/아티팩트/하우징/상점) |
| _Story | 시나리오 재생 |
| _InGame | 전투 (퍼즐 + 디펜스) |
| _Test | 밸런스 테스트 전용 — 스테이지 직진입, 성장 레벨 시뮬, 수치/배속 조절 |

## 실행 방법

- **에디터**: `Scenes/_Boot`을 열고 재생 (정식 플로우) 또는 `_InGame`/`_Test` 단독 재생 (자가 조립되어 바로 전투)
- **밸런스 테스트**: `_Test` 씬 재생 → 화면 중앙 상단 TEST 버튼 → 스테이지 선택/수치 조절 (진행도·재화에 기록되지 않는 격리 환경)
- **데이터 수정**: 시트 수정 → JSON 내려받기 → 에디터에서 재파싱(Import) → 재생
